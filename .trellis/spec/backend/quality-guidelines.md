# Quality Guidelines (C# / Grasshopper)

> Code quality criteria and compliance checklists for Motion codebase.

---

## 1. Grasshopper Component Requirements

Every custom component MUST satisfy the following structural requirements:

1. **Unique ComponentGuid**: Every component class must define a unique `Guid` generated by standard GUID generators. Never copy-paste GUIDs between components.
2. **Icon Resource**: Every component must override the `Icon` property, returning a `System.Drawing.Bitmap` resource.
3. **GH_Exposure**: Define the category exposure (`GH_Exposure.primary`, `secondary`, etc.) to arrange the component in the toolbar tab.
4. **SolveInstance Safety**: Always validate inputs first before calculating. Return early if critical data is missing or null.

---

## 2. Naming Conventions

- **PascalCase**: Class names, public methods, properties, events.
- **camelCase**: Parameters, local variables.
- **Private Fields**: Prefix with an underscore `_` (e.g., `_cancellationTokenSource`).
- **Category Folders**: Organised by category prefix like `01_Animation`, `02_Export`, `03_Utils`.

---

## 3. Pre-check Checklist

Before committing modifications:
- [ ] Clean build passes for all target platforms (`net7.0-windows` / `net48`).
- [ ] Unused resources/assets are removed or marked obsolete.
- [ ] All external resources/assemblies load properly at runtime.
- [ ] Disposal lifecycles are implemented for components using unmanaged handles.
